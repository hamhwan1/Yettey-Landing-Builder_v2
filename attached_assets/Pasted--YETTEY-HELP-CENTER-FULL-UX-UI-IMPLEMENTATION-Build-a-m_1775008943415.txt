# 🔥 DETAIL PAGE STRUCTURE (VERY IMPORTANT)

Each Quick Action / FAQ item must open a structured help page:

---

## Page Layout

1. Title (problem-based)
   Example:
   Upload failed

---

2. Short explanation
   Explain what the issue means in 1–2 lines

---

3. Step-by-step solution

Example:

Step 1
Check your file format (MP4, MOV supported)

Step 2
Ensure file size is under limit

Step 3
Retry upload

---

4. Visual support (if possible)

* Screenshot
* UI highlight

---

5. Alternative solution

Example:
If the issue continues:

* Refresh the page
* Try another browser

---

6. Related articles

Example:

* Video processing stuck
* Supported file formats
* Upload limits

---

7. CTA

Still not working?

[ Contact Support ]

---

# UX PRINCIPLES (MANDATORY)

* Focus on problem-solving, not explanation
* Reduce reading → increase clarity
* Make navigation extremely fast
* Use clear, simple language
* Every page must lead to resolution

---

# DESIGN NOTES

* Use card-based UI for quick actions
* Keep spacing clean and minimal
* Highlight search as primary action
* Ensure mobile-friendly layout

---

# FINAL GOAL

Users should:
→ Find answers instantly
→ Solve issues without contacting support
→ Understand the product naturally
